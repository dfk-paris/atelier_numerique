A very short introduction

Here we will use the Wikidata Query Service to get a first impression of SPARQL and RDF. SPARQL is the language one uses to get data that is stored as RDF. Essentially this means data that is stored in graphs instead of relational databases (like MSAccess oder Filemaker). This graphs are made of subject-predicate-object statements like

subject: Fatima     predicate: has birthdate              object: 26.12.1986 
subject: Fatima     predicate: has birthplace             object: Kairo
subject: Nina       predicate: is biological parent of    object: Fatima
subject: Nina       predicate: has Gender                 object: Female

This strucuture allows us to find implicit informations like that Nina stayed at the 26.12.1986 in Kairo, as she gaves birth to her daughter Fatima.
(Btw. the automatic axtraction of this biographic informations is the main object of the project Cosmotool (https://cosmotool.de.dariah.eu/cosmotool/personsearch/)).

This is only a glimpse on a huge topic and we leave it for now. If you continue on your own with the Wikidata Query service, it will help you to have an idea about the structure as you built the questions. So at this point the main take away is: If you can talk SPARQL you can find Data at places like the {Bnf (https://data.bnf.fr/current/sparql.html), Europeana (https://pro.europeana.eu/page/sparql) and many more, that offer a SPARQL-endpoit. So far so nice, but it is part of the deal, that it takes some time to get fluent in SPARQL and that you need to know the database to which you are sending your request. Among others Wikidata fortunately offers a GUI (graphical user interface) that allows you to test and explore SPARQL on the immense variety and richness of the Wikidatabase. There is also a mass of SPARQL requests as examples, so there is plenty of material to test.

Let’s start

The first request (and some cat-content)

Let’s jump to the Wikidata Query service https://query.wikidata.org/

Here you find under “examples” - at the very top - one that is entitled “cats” and is the first one.

Voilà: the first SPARQL-Query

Let’s just run the query by pressing the blue framed button at the left of the main window. Afterwards you will see below a table with 144 items (as of 08/05/2020). You can download this table in several different formats like csv and JSON. You can also click on one of the numbers in the first column.

As interesting as all this is, we still do not know exactly what this table is about. So let us break down the query.

#Cats
SELECT ?item ?itemLabel
WHERE
{
?item wdt:P31 wd:Q146.
SERVICE wikibase:label { bd:serviceParam wikibase:language “[AUTO_LANGUAGE],en”. }
}

step by step

  • #Cats This is a just a comment of the one, who provided this question. The machine skips everything that follows the hash (#).
    (Btw. It’s the same how you use <!-- for just another comment --> in HTML.)

  • SELECT ?item ?itemLabel

    • By SELECT we determine the categories of the result, or what kind of information we want to get. In this case we want to get each cat, that made an entry to Wikidata as an item, and their names. So we are asking for “?item” to have a list of all items and for ?itemLabel" to add their names.

  • WHERE { This is there the fun starts, as we are describing what kind of items we are looking for and asking for adding ttheir names

    • ?item wdt:P31 wd:Q146. Now we are asking for items or individuals who are “instances of”(wdt:P31) “house cats” (wd:Q146). Well, this is an important piece to understand. There exist “concepts” and “representatives” of these concepts. Like the concept “continent” and “Asia, Africa, Europe” as some instances or realisations of “continent”. It’s the same with “animal” and “elephant, lion and nemo”. Back to our query, here we are asking: “?item” = give me all the datasets that meets the condition, that they are wdt:P31 an instance of wd:Q146 house cat.

    • SERVICE wikibase:label { bd:serviceParam wikibase:language “[AUTO_LANGUAGE],en”. } by this we are asking for the labels of each item, and as quite often there are different labels for one item depending on the language. You can ask for an english label, or if there is no english label for the default. Just think of the item Africa, which has the french label “Afrique”, the german label is “Afrika” and, well just have a look yourself: https://www.wikidata.org/wiki/Q15.

    • finally “}” Well, whenever you open a bracket you need to close it. Take it as a mantra.

Let’s have some more fun and add some pics

Wouldn’t it be fun to have a look at those cats?
To add them, firstly we are asking for SELECT ?item ?itemLabel (and new) ?itemImage, secondly we add a further constraint: as the ?item should not only be an instance of housecats but the item should have an image (wdt:P18) that goes as ?itemImage. The rest stays the same.

SELECT ?item ?itemLabel ?itemimage
WHERE
{
?item wdt:P31 wd:Q146.
?item wdt:P18 ?itemimage.
SERVICE wikibase:label { bd:serviceParam wikibase:language “[AUTO_LANGUAGE],en”. }

}

  • The resulting table has a third column with image-links. On the left next to the table there is a dropdown menue and you can change from “table” to “image grid” to see the cats (a bit weird actually).

  • Well, here are two more interesting observations.

    • The first one is about properties. Currently we are asking those items, that are instances of something, to have certain properties. So, let’s think of wine as an instance of beverages and red and white as properties of the item wine. In our cat-content-example some of the items fullfil the property to offer an image. Most of the instances of house cat do not fullfill this, that’s why or list of results is rather short with only 36 results.
    • The second observation is less conspicuous. It is about the terms we use to name the outputs. ?item ?itemLabel and ?itemImage are interchangeable. Although not without respecting the dependencies. So you can replace ?item by ?WikidataItem and ?itemImage with ?Pic. Runing this will result in fieldnames of the table, that may be better understandable. Now, you can replace ?itemLabel with ?Name. But after runing this the column will be empty. This is because you broke uninentionally a dependency of of class and instance of and the connection to the SERVICE wikibase:label. Replacing ?Name with WikidataItemLabel will fix the Problem. (And if you want to dig deeper: https://www.mediawiki.org/wiki/Wikidata_Query_Service/User_Manual#Label_service)

SELECT ?WikidataElement ?WikidataElementLabel ?Pic
WHERE
{
?WikidataElement wdt:P31 wd:Q146.
?WikidataElement wdt:P18 ?Pic.
SERVICE wikibase:label { bd:serviceParam wikibase:language “[AUTO_LANGUAGE],en”. }

}

  • What if we want to have all the cats again, and if someone added an image to their Wikidata Element, we would like to have them too. That is easy as we could describe the existence of an image as an optional property. So we just insert: OPTIONAL {?WikidataElement wdt:P18 ?Image.}

SELECT ?WikidataElement ?WikidataElementLabel ?Pic
WHERE
{
?WikidataElement wdt:P31 wd:Q146.
OPTIONAL {?WikidataElement wdt:P18 ?Pic.}

SERVICE wikibase:label { bd:serviceParam wikibase:language “[AUTO_LANGUAGE],en”. }

}

One more cat-content nonsense

  • Okay, now if you are inclined, we can do one more weird thing, and that is to ask, which of the cats have an own entry at the WorldCat ID.

    • This is again a Property of the item, so we have to ask for a property §. To identify the signifier (numer) of the property, just open Wikidata in a new table and search for WorlCat ID. You may find https://www.wikidata.org/wiki/Q76630151. This may look at first glance convincing, but will not work. As Q76630151 covers all IDs of the worldcat, each single ID is something starting with a P as the individual ID again is a property. You will find this within the ontological description of Q76630151 (aka. on the page).
    • Insert a phrase above or below the OPTIONAL-constraint. The phrase should follow the example ?WikidataElement wdt:P18 ?Pic, but is not meant as an option but an obligation. Remember to insert a variable (a fieldname) in the SELECT

  • Okay, and if you aren’t fed up with cats, just try to find out which of those WorlCat-cats even have a Twitteraccount (Twitter Username).

More seriuos stuff

Maybe you want to move on to something more serious.

  • By this query we are asking for all those places in France, there more than five female writers died. The result should be orderd by the placenames.

SELECT ?dplace ?dplaceLabel (COUNT (?author) AS ?deaths)
{
?author wdt:P31 wd:Q5.
?author wdt:P21 wd:Q6581072.
?author wdt:P106 wd:Q36180.
?author wdt:P20 ?dplace.
?dplace wdt:P17 wd:Q142.

SERVICE wikibase:label { bd:serviceParam wikibase:language “[AUTO_LANGUAGE],en”. }
}
GROUP BY ?dplace ?dplaceLabel
HAVING (?deaths > 5)
ORDER BY ASC(?dplace)

  • Try to change this to a less morbid topic and ask the same but in reference to places of birth (as GROUP BY can be tricky, be patient and exact)

  • Try to get rid of the grouping and instead line up all the writers, with their names and their birthplaces.

  • Try to add images of the birthplaces.
